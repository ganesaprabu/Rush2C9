# Rush2C9 - Hackathon Submission Checklist

## Sky's the Limit - Cloud9 x JetBrains Hackathon

### Before Submission Day

- [ ] Complete all core game features
- [ ] Test on mobile devices
- [ ] Fix any critical bugs
- [ ] Update documentation (README, PROGRESS.md)

---

### On Submission Day

#### 1. GitHub Repository
- [ ] **Make repository PUBLIC** (Settings ‚Üí General ‚Üí Danger Zone ‚Üí Change visibility)
- [ ] **Add/Verify LICENSE file** (Recommended: MIT License for hackathons)
- [ ] Ensure README.md has:
  - [ ] Project description
  - [ ] Demo link / screenshots
  - [ ] How to run locally
  - [ ] Tech stack used
  - [ ] Credits / acknowledgments

#### 2. Live Demo
- [ ] Deploy to hosting (Vercel/Netlify/GitHub Pages)
- [ ] Test the live demo link works
- [ ] Test on mobile browser

#### 3. Devpost Submission
- [ ] Project name and tagline
- [ ] Detailed description
- [ ] Demo video (if required)
- [ ] Screenshots/GIFs
- [ ] Link to GitHub repo (must be public!)
- [ ] Link to live demo
- [ ] List technologies used
- [ ] Team members

#### 4. Final Verification
- [ ] Play through entire game once on live demo
- [ ] Verify all 3 segments work
- [ ] Check celebration/confetti displays
- [ ] Test on different devices if possible
